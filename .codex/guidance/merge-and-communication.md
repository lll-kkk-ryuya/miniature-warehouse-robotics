# Merge And Communication

Source reference: `.claude/rules/merge-and-communication.md`.

- Use GitHub Issues and PRs as the recorded communication channel.
- Put the worktree tag at the top of handoff comments and PRs.
- Do not self-merge in the same turn as PR creation.
- PRs should be visible, testable, and reviewable before merge.
- After merge, remove stale worktrees and branches.
