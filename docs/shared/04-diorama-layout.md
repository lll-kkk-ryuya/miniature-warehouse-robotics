## エリア定義

| エリア | 役割 | サイズ目安 |
|--------|------|----------|
| バースA/B | トラック接車・荷受け | 各200×150mm |
| 棚1〜3 | パレットラック（保管） | 各150×300mm |
| 通路A/B | 棚間の搬送路（縦方向） | 幅200mm |
| 通路C | 横断通路 | 幅200mm |
| 出荷ステーション | 出荷待ちエリア | 200×150mm |
| 充電ステーション | ロボット待機・充電 | 150×150mm |

## 通路幅の設計

| 要件 | 値 |
|------|-----|
| ロボット幅 | 約150mm（※Yahboom実測で要確認） |
| 最小通路幅（一方通行） | 180mm |
| 標準通路幅（すれ違い可） | 300mm |
| 渋滞を意図的に発生させる通路 | 200mm（すれ違い不可） |

Before/After のデモ効果を出すため、**通路A/Bは200mm（すれ違い不可）に設定**する。
After ではLLM司令官が一方通行ルール・迂回・タイミング調整を指示して渋滞を解消する。

## ミッションシナリオ（2台構成）

### シナリオ1: 通常搬送

1. Bot1: バースA → 棚1 → 出荷ステーション
2. Bot2: バースB → 棚2 → 出荷ステーション

→ 通路Aで Bot1 と Bot2 が鉢合わせ → 渋滞

### シナリオ2: 障害物出現（デモの見せ場）

1. Bot1, Bot2 が搬送中
2. 人の手でジオラマに障害物（小さな箱）を配置
3. LLM（Claude等）が状況を判断し指示を出す:
   - Bot1: 通路B経由で迂回
   - Bot2: 3秒待機 → 衝突回避

→ LLMのリアルタイム判断力を実演

### シナリオ3: 動的タスク割り当て

1. 3つのタスクが同時発生（棚1→バースA、棚2→出荷、棚3→バースB）
2. LLMが距離・現在位置を考慮して最適割り当て
3. 先に終わった方が残りのタスクを自動で取る

→ LLMの戦略判断力を実演

### Before/After の定義

| モード | 動作 |
|--------|------|
| Before（LLMなし） | 各ロボットが独立して最短経路で走行（渋滞・衝突が発生） |
| After（LLM司令官） | LLMの判断に従い、迂回・タイミング調整・タスク再割当を実行 |

## 構築手順

1. ラワン合板をテクスチャーペイントで塗装（コンクリート床風）
2. ビニールテープで通路境界・マーカーを貼る
3. 3Dプリントした棚を配置（接着せず、置くだけ → レイアウト変更可能）
4. バース・出荷ステーションを配置
5. minicar搭載ORBBEC MS200でteleop走行 → SLAM Toolboxで2D地図生成
6. Nav2のコストマップに反映

## 重要な前提条件

**通路幅の設計はロボット実寸に依存している。**
現在のロボット幅（約150mm）は未検証であり、実寸がこれより大きい場合、Before/After デモの渋滞シナリオが成立しない可能性がある。Phase 1 でロボット到着後、必ず実測してからレイアウトを確定すること。
