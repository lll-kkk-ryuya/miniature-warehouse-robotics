# YouTube映像構成（ストーリーボード）

作成日: 2026-05-21
更新日: 2026-05-23
形式: YouTube公開動画（5-7分）
公開区分: 公開（合成データ・仮想レイアウト・一般論のみ）

## 映像の目的

「Claude vs ChatGPT — AIに倉庫ロボットを運転させてみた」

LLMが倉庫ロボット2台の司令官としてリアルタイムで判断する様子を見せる。
複数LLMで同じシナリオを実行し、判断の質・速度・効率を比較する。

## 営業での使い方

| 用途 | 効果 |
|------|------|
| 商談前に送る | 「AI×物流の実機デモを持っている会社です」→ 信頼形成 |
| ヒアリング獲得 | CTA「御社の倉庫でもAI診断してみませんか？」 |
| 商談中のデモ | 「これを御社のデータでやると…」→ 診断提案に繋げる |
| SNS/YouTube | AIがロボットを運転 + LLM比較 → 拡散力が高い |

## 映像設計思想

### 実機映像が主役、Isaac Simは補助

本動画の映像は**実機（ジオラマ上のロボット）の俯瞰カメラ映像が主役**である。Isaac Simの3D CG映像は「デジタルツインでも再現できる」という技術力アピール用の補助素材であり、動画全体の約5%（30秒程度）にしか使用しない。Isaac Simが間に合わない場合でも、実機映像だけで動画として成立する設計。**※例外: 実機未到着での「先行リリース版」は Gazebo 俯瞰録画を主素材とする（→ 後述「## 先行リリース構成（実機未到着・sim録画版）」。これは本番＝実機主役の前提を置き換えず、実機完成後に本番版へ差し替える暫定経路）。**

### 映像素材の役割分担と使用割合

| 映像素材 | 使用時間 | 割合 | 役割 |
|---|---|---|---|
| **実機映像**（俯瞰カメラ） | 約4分 | **70%** | 動画の主役。ロボットが実際に動いている証拠映像 |
| LLM思考ログ・KPIグラフ | 約1.5分 | 25% | AIが何を考えて指示したかの可視化。視聴者の理解を助ける |
| **Isaac Sim 3D映像** | **約30秒** | **5%** | デジタルツインの技術デモ。「実倉庫スケールだとこう見える」というスケール感の補完 |

### なぜ Isaac Sim の映像を足すか

実機の俯瞰映像だけだと「15cmのロボットが合板の上を動いている絵」になる。Isaac Simのフォトリアルな3D映像を30秒だけ横に並べることで、以下の効果を狙う。

| 観点 | 実機映像だけ | 実機 + Isaac Sim |
|---|---|---|
| 視聴者の印象 | 「ミニチュアだな」で終わる | 「実際の倉庫でもこう動くんだ」と想像させる |
| 営業効果 | 技術デモ感 | **プロダクト感・将来性**を演出 |
| 競合との差別化 | ROS 2 + LLM のデモ | ROS 2 + LLM + **NVIDIAデジタルツイン**のフルスタック |

### Gazebo と Isaac Sim の使い分け

| | Gazebo Harmonic | Isaac Sim 5.1 |
|---|---|---|
| **誰のための映像か** | 開発者（自分） | 視聴者・営業先 |
| **用途** | Phase 0.5 でソフト開発・テスト | Phase 5 でYouTube用の3D映像生成 |
| **映像品質** | 低〜中（開発には十分） | **フォトリアル**（レイトレーシング） |
| **動作環境** | Mac Docker で動く（無料） | RTコア必須GPU — RunPod A10G $0.37〜0.54/h |
| **動画への登場** | 登場しない（開発用。※実機未到着時の「先行リリース版」のみ主素材として登場＝§先行リリース構成） | 5:00-5:30 の30秒間 |

Gazeboは「コードが正しく動くかを検証する開発ツール」、Isaac Simは「YouTube視聴者に見せるための映像生成ツール」。役割が全く異なる。

## 画面構成

動画全体を通して、シーンごとに異なる画面レイアウトを使い分ける。1つの画面を分割して複数の情報を同時に見せる構成であり、「別々の動画を横に並べる」のではなく、**1つの動画内で画面を区切る**。

### パターンA: メイン画面（2:00-3:30, 3:30-4:00 で使用）

LLM制御シーンの標準レイアウト。実機映像 + 思考ログ + 地図を同時表示し、「ロボットが何をしているか」「AIが何を考えているか」「地図上でどう動いているか」を一画面で伝える。

```
┌─────────────────────┬──────────────────────┐
│                      │  LLMの思考ログ        │
│   俯瞰カメラ映像     │                      │
│  （実機ジオラマ）     │  「Bot2の前方に障害物  │
│                      │   を検出。通路Bへ     │
│   🤖→  ■  🤖        │   迂回を指示。Bot1は  │
│                      │   3秒待機させて衝突   │
│                      │   を防ぐ」            │
├─────────────────────┤                      │
│  RViz / Foxglove     │  commands:           │
│  （地図＋経路表示）    │  bot2 → route_B      │
│                      │  bot1 → wait 3s      │
└─────────────────────┴──────────────────────┘
 ← 左側: 視覚情報 →    ← 右側: AI判断情報 →
```

| 画面領域 | 内容 | 情報の役割 |
|---|---|---|
| 左上（約50%） | 俯瞰カメラの実機映像 | 「実際にロボットが動いている」ことの証拠 |
| 左下（約15%） | RViz/Foxgloveの地図画面 | ロボット位置・計画経路・コストマップの技術的可視化 |
| 右側（約35%） | LLMの思考ログ + コマンド | AIが何を考え、どんな指示を出したかのリアルタイム表示 |

### パターンB: フル画面（0:00-2:00 で使用）

Before（AIなし）シーンや導入部では、実機映像をフル画面で見せる。分割なし。

```
┌────────────────────────────────────────────┐
│                                            │
│            俯瞰カメラ映像                    │
│           （実機ジオラマ全景）                │
│                                            │
│          🤖→        ←🤖                    │
│              通路で鉢合わせ！                │
│                                            │
└────────────────────────────────────────────┘
```

### パターンC: LLM比較・左右並列（4:30-5:00 で使用）

同じシナリオをClaude vs ChatGPTで並べて見せる。同じ時点の実機映像を2つ横に並べ、判断の違いが一目でわかるようにする。

```
┌────────────────────┬────────────────────┐
│                    │                    │
│   Claude編         │   ChatGPT編       │
│  （実機映像）       │  （実機映像）      │
│                    │                    │
│  🤖→ 迂回成功      │  🤖→ 3秒迷う      │
│                    │                    │
├────────────────────┼────────────────────┤
│ 応答: 1.2秒        │ 応答: 1.8秒       │
│ 判断: 迂回指示      │ 判断: 停止→再計画  │
└────────────────────┴────────────────────┘
```

### パターンD: デジタルツイン並列（5:00-5:30 で使用 — 30秒のみ）

実機映像とIsaac Simの3D CG映像を横に並べる。同じロボットの動きが「ミニチュア」と「リアルな倉庫CG」で同時に見え、スケール感を補完する。このパターンは動画中で**1回だけ、30秒間**使用する。

```
┌────────────────────┬────────────────────┐
│                    │                    │
│   実機映像          │   Isaac Sim        │
│  （俯瞰カメラ）     │  （3D CG映像）     │
│                    │                    │
│  ミニチュア倉庫     │  リアルな倉庫CG    │
│  🤖 15cmロボット    │  🤖 実寸ロボット   │
│                    │                    │
└────────────────────┴────────────────────┘
 ← 「今やっていること」  「実倉庫だとこう見える」→
```

| 要素 | 左（実機） | 右（Isaac Sim） |
|---|---|---|
| 倉庫 | 1.8m×0.9m合板ジオラマ | 同じレイアウトのフォトリアル3D倉庫 |
| ロボット | Yahboom minicar（15cm） | 実寸スケールの3Dロボットモデル |
| データ同期 | ROS 2トピックで位置送信 | ROS 2 Bridgeで受信→同じ動きを再現 |
| 視聴者への効果 | 「ミニチュアで動いている」 | 「実倉庫でも同じように動く」 |
