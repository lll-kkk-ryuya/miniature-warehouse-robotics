LLM支援 Rule Authoring 図解 — Productization

顧客が自然言語で出す現場 rule を、LLM が draft し、人間 review・fixture・simulation / eval gate を通して、site profile / plugin profile / run manifest へ昇格する流れを図解する。runtime enforcement は L3 / L2 / L1-L0 の deterministic box が担当する。

12. Non-Goals / Safety Boundary

  • LLM が runtime policy を直接実行する仕組みは作らない。10:207-210
  • Handoff に顧客別 rule を混ぜない。10:209-210
  • L2/L1/L0 safety enforcement を Validator plugin で置換しない。10:211
  • 未凍結の coordinate goal、3D collision check、safety-rated distance を product contract として扱わない。10:212-213
  • Command Compiler は gen_id、idempotency_key、velocity、low-level action、未凍結 coordinate goal を作らない。03:167-172
