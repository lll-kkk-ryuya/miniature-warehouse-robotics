7. 二段ゲート と L2 restrict-only policy profile（2026-07-09 思想・未実装）

L3 Validator と L2 Policy Gate は似て見える check の重複ではない＝同一事故を別入力・別時刻で止める二段。L3 は「Command 候補になってよいか」を plan 時に、L2 は「今この tool call を実行してよいか」を dispatch 時に判定する。正本 = 11 §2026-07-09 補足 ＋ ADR-0004。

L3 Validator（問い「Command 候補になってよいか」）
入力 = LLM / ER の plan draft。失敗 → tool call を組み立てない。語彙 = 凍結 9-code ValidationCode（UNKNOWN_ROBOT / UNKNOWN_TARGET / EMERGENCY_ACTIVE / CYCLE_STATE_STALE …）。report.py:69-88
▼
L2 Policy Gate（問い「今この tool call を実行してよいか」）
入力 = dispatch 引数 ＋ live StateSnapshot。失敗 → Nav2 へ forward しない。語彙 = 14 policy_gate code（§1.2）。policy_gate.py / §1.2
紛らわしい 3 対（別層・別語彙・別時刻）: UNKNOWN_TARGET〔L3〕 vs missing_location/unknown_location〔L2〕 ／ EMERGENCY_ACTIVE〔plan 時〕 vs robot_in_emergency〔dispatch 時〕 ／ CYCLE_STATE_STALE〔cycle 単位〕 vs robot_stale 0.5s〔per-robot〕。defense-in-depth: L3 が stale / bug でも L2 が止める（L3 は実行権限なし・L2 は planning 知能なし）。report.py:69-88 / policy_gate.py
accepted-motion forward は今や 3-way config gate: L3 を通った候補が Policy Gate 受理後に Nav2 へ抜けるかは構成依存 — Mode A/B = forwarder 配線 ／ Mode C = forwarder=None（0 actuation・R-26）／ X-ER = config mode_x_er.dispatch.forward_to_nav2（safe-OFF 既定 None = 0 actuation・true → Nav2RestForwarder・#423）。「Mode A/B のみ」ではない。tools.py _maybe_forward / mode_x_er.dispatch.forward_to_nav2
