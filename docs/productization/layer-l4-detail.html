0. Super-Box の全体構造図（I/O・sub-box・seam・demoted の入れ子）

L4 Super-Box は1つの差替単位（box）でありながら、内部に sub-box・seam・demoted を入れ子で所有する。下図は「箱の中に何がどう入っていて、どこから外（Governance）へ抜けるか」を一枚で示す（各要素の細部は §3、サイクルの時系列は §1、排他制御は §2）。02 §L4 の責務 / box-taxonomy §2

box（契約を変えず差替/省略できる再利用単位）
sub-box（親内部ステージ・親ごと差替）
seam（境界の写像・常時随伴で省略不可）
transport / plugin（interface 裏の実装選択）
demoted（box でない＝Super-Box へ吸収）

2. 排他制御 3層（A + B-3 + C）

同時発火・stale・replay を3層で塞ぐ。この3層は Bridge-owned の安全核で Hermes へ移譲しない。doc08:161-179 / doc15 §2

A
client-side cancel（asyncio.wait_for(2.5s)）。新サイクルが古い LLM 呼び出しを cancel。明示 /v1/runs/{id}/stop は #54 で撤回（stateless chat/completions + in-process dispatch にサーバー側 run 無し）。安全主担保は B-3+C。scheduler.py:309 / doc08:140,173-179
B-3
同世代ガード。gen_store.set で世代公開 → action_map が全 ToolCall に gen_id 注入 → MCP GenChecker が stale gen を reject。gen_store / action_map.py / gen_check.py GenChecker.check
C
replay ガード。action_map が per-call idempotency_key=str(uuid.uuid4()) を mint → MCP check_and_add で同一 call の replay を reject。key は Bridge mint・LLM 由来不可（"minted here, NOT read from LLM-controlled CommandItem.idempotency_key"）。action_map.py:5-9,43 / gen_check.py
安全の非対称性: LLM に「一意で繰り返さない UUID を出せ」と信頼しない。idempotency_key は Bridge が send 時に mint する（action_map.py:7-9）。これが「Hermes に寄せない」核心（box の真ん中を Hermes 線で割らない）。

3. sub-box と seam の細部

Input Context sub-box（実装あり）

situation.py SituationBuilder。mode-aware: Mode A/B は predicted_position_3s（CTRV 等速・等旋回）＋ obstacle_ahead（<emergency_min_distance=0.3）含む全フィールド。Mode C(open-rmf) は velocity/heading/predicted/obstacle を構築せず省略（~200tok 節約）。current_task(bot→destination) と pending_tasks（WAREHOUSE_TASKS seed・PendingTask with alias from）を両モードで供給。consumes StateSnapshot → produces 凍結 Situation。situation.py / 06 §L4 Input Context / mode-a/08a

Model Transport / Adapter sub-box ER/VLA/STT は未実装＝proposal

現行 commander LLM の Hermes transport は hermes_client.py で実装済み。ER/VLA/STT の transport registry は proposal。transport: hermes | direct | worker は interface 裏の実装選択（box を割らない）。raw output recorder・timeout・0 dispatch は Bridge-owned。hermes_client.py / 02 §Box の保管場所 / mode-x-er/03-er-adapter-skeleton

Fusion sub-box optional・未実装

ER/VLA/STT/WMS が食い違ったときの arbitration（disagreement policy・confidence fusion・operator clarification）。単一 model では使わない＝抜ける。VLA output は直接実行せず必ず Fusion/L3 を通す。02 §VLA adapter / 06 §L4 Fusion / mode-x-er-vla/03

action_map / MCP dispatch seam（実装あり）

action_map: command_item_to_tool_call(item, gen_id, idempotency_key?) が Command(list[CommandItem]) を MCP ToolCall へ。CommandAction=navigate/wait/stop/yield/charge（schemas.py CommandAction）→ dispatch_task / cancel_task / send_to_charging。MCP dispatch: DispatchToolExecutor(tools.dispatch)（executor.py:35 / llm_bridge.py:24-26,207）。Hermes の MCP tool transport(stdio/HTTP) はこの seam 裏の実装。server-side motion 即時実行は不可。01 §境界の原則 / architecture/15-mcp-platform
