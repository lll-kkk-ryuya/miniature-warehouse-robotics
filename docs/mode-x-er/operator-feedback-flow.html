4. 発話スコープ — 鳴り続けない工夫（命令に紐づく節目だけ）

L2〜L0 は operator 命令と無関係にも止まりうる（自律 collision stop・battery 自律帰還・相手への一時 yield・50ms tick…）。全部喋ると鳴り続けるので、喋るのは「命令(gen_id=N)に紐づく失敗・節目」だけ。05 §5.3

speak ⟺ event.gen_id ∈ {live な operator 命令} ∧ decision ∈ {rejected, needs_clarification, emergency_stop,（任意で arrived/completed）} ∧ lifecycle transition（高頻度 tick ではない）

🔊 喋る（命令起因・実アクションの節目）

deadlock 解消不能 / no_path / recovery_exhausted / target 未解決 / battery 不足で命令不可 / 到着(completed) / 右折など task の milestone。

🔇 黙る（命令外・高頻度・無関係）

命令外の自律 collision stop / 相手への一時 yield（命令は継続中）/ 50ms 安全 tick / Eval 集計用の routine sample。＝web/log のみ。

「私の命令が L2〜L0 で止まったら？」の処理: ① 命令は L3 で Command(gen_id=N) に compile → action_map が gen_id=N を注入して dispatch。② 下流で その命令が止まると、止めた箱が gen_id=N 付き decision_event を emit（非ブロッキング）。③ ★ が「gen_id=N は自分の live 命令か?・節目か?」で filter → yes なら音声化。④ 命令と無関係な停止（gen_id 無 / autonomous / tick）→ 除外 → 無音（web/log のみ）。紐付け鍵は funnel と同じ gen_id/run_id。05 §5.3 / 02-l3:178-182
