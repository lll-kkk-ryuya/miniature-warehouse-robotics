Mode X-ER 実装状況 — 想定(planned) vs 現状(current on main 4114195)

ER → L3 パイプラインの各 node を、想定(planned) と 現状(main 4114195 に載っているもの)で対比する。結論から言うと、凍結 Command まで到達する offline L3 チェーンは DONE（envelope → handoff → validator → resolver（object-class snap 込み #452）→ executor → compiler → pipeline.compile_raw_output）、LIVE send 機構（#389）も main 着地、さらに それらを鎖化する背骨 x_er_bridge（#419/#423）が land＝goal_result だけで red→blue を offline 完走。ただし 実 rclpy node を sim で spin し L2/L1/L0 を実作動させる XER6 sim closure は G5 human gate＝未実走（#456 の無償 replay 経路 mode_x_er.er_offline_payload land 済＝課金なしで実走可）。ER transport の標準は fork gateway（8644）一本で全 modality を Hermes 経由（ADR-0002・live-matrix 12/12 sends 実証）、shipped default の audio は依然 DIRECT（下の紫バンド）。本ページは図解（補足資料）で、設計の正本は 01〜07 / README の .md。

つながり（connectivity）— 上から running で届くか

STATUS（塗り）とは直交する第 2 軸。「稼働している rclpy node から、上（operator 入力）→下（motor）へ各 seam に到達できるか」。答え（2026-07-07・#419/#423 land 後）: 欠けていた 1 ピース＝X-ER commander node x_er_bridge が landし、hops ⓪(稼働 cycle 部分)②③④⑤ を鎖化する caller が揃った。ただし正直な区別: OFFLINE-WIRED（node 実装・offline pytest で cycle が走り red→blue 完走）≠ RUNNING（実 rclpy node を sim で spin し L2/Nav2 まで実作動）。後者は G5 human gate＝まだ実走していない（launch は mode_x_er.enabled∧llm gate・base 既定 OFF）。実マイク capture（hop⓪）も未実装。
